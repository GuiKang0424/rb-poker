## 项目概述

### 项目目标
创建一个基于 Web 的多人德州扑克游戏，支持真人玩家与 AI 玩家同局对战，采用实时 WebSocket 通信，保证游戏的公平性和流畅性。

### 游戏模式
- **核心玩法**：无限注德州扑克（No Limit Texas Hold'em）
- **单桌人数**：最多 10 人
- **玩家类型**：真人游客 + 规则型 AI
- **AI 风格**：5 种不同策略风格
- **房间机制**：房主创建房间，自定义盲注级别

### 游戏状态机设计
```
                      ┌─────────────┐
                      │   Waiting   │  等待玩家加入
                      └──────┬──────┘
                             │  人数满足，开始新局
                      ┌──────▼──────┐
                      │   Preflop   │  发底牌，从小盲开始下注
                      └──────┬──────┘
                             │  下注轮结束
                      ┌──────▼──────┐
                      │    Flop     │  发 3 张公共牌，下注轮
                      └──────┬──────┘
                             │  下注轮结束
                      ┌──────▼──────┐
                      │    Turn     │  发第 4 张公共牌，下注轮
                      └──────┬──────┘
                             │  下注轮结束
                      ┌──────▼──────┐
                      │    River    │  发第 5 张公共牌，下注轮
                      └──────┬──────┘
                             │  下注轮结束
                      ┌──────▼──────┐
                      │  Showdown   │  亮牌，比大小，分配筹码
                      └──────┬──────┘
                             │  牌局结束
                      ┌──────▼──────┐
                      │   Waiting   │  等待下一局开始
                      └─────────────┘
```

### 状态同步机制
1. **后端唯一真理源**：所有游戏逻辑、状态计算、规则判定全部在后端执行
2. **事件驱动**：后端只向前端推送游戏事件（如 "player_bet"、"card_dealt"）
3. **动作确认**：每个前端玩家动作需后端 ACK 确认后才生效
4. **非法动作拒绝**：不符合当前状态的动作直接 reject 并返回错误

### WebSocket 事件协议
**服务端 → 客户端（S2C）事件：**
```typescript
// 玩家加入
{ type: 'player_join', data: { seat, nickname, chips } }

// 玩家离开
{ type: 'player_leave', data: { seat } }

// 游戏状态变更
{ type: 'state_change', data: { state, dealerSeat } }

// 发底牌（只有对应玩家能看到自己的牌）
{ type: 'hole_cards', data: { seat, cards } }

// 发公共牌
{ type: 'community_cards', data: { cards, stage } }

// 玩家动作
{ type: 'player_action', data: { seat, action, amount } }

// 牌局结束
{ type: 'hand_end', data: { winners, pots, showdownHands } }
```

**客户端 → 服务端（C2S）动作：**
```typescript
// 加入房间
{ action: 'join_room', data: { roomId, nickname } }

// 游戏动作
{ action: 'fold' }
{ action: 'check' }
{ action: 'call' }
{ action: 'bet', data: { amount } }
{ action: 'raise', data: { amount } }
{ action: 'all_in' }

// ACK 确认
{ action: 'ack', data: { actionId } }
```

## 开发原则

1. **后端优先**：所有游戏逻辑在后端实现，前端只做渲染
2. **类型安全**：前后端都用类型系统（TypeScript + Pydantic）
3. **可测试**：核心游戏逻辑写单元测试
4. **渐进式开发**：按里程碑迭代，每个里程碑都有可运行版本
5. **防作弊**：后端权威状态机，不相信前端任何数据
