# 牌局聊天室 — 详细设计方案

> 2026-06-25。

## 1. 概述

为房间内的所有玩家（等待区 + 已入座）提供实时聊天功能，支持文本聊天、快捷短语和系统消息。右侧面板展示，不遮挡牌桌核心区域。

## 2. 核心决策

| 维度 | 决策 | 原因 |
|------|------|------|
| 聊天范围 | 房间内（等待区 + 入座玩家） | 扑克社交核心在牌桌，围观玩家也能互动 |
| 消息类型 | 文本 + 快捷短语 + 系统消息 | 系统消息增强氛围，快捷短语降低打字门槛 |
| 持久化 | 内存缓存最近 100 条，不写 DB | 新玩家加入补发历史融入氛围；零持久化成本 |
| 限频 | 不限 | 娱乐场景，信任用户；后期若需要可加 1s/3条 冷却 |
| 内容过滤 | 不做过滤 | 前端 HTML encode 防 XSS；熟人房间制不涉及公共论坛监管 |
| UI 位置 | 右侧可收起/展开面板 | 不遮挡牌桌核心区域 |
| 快捷短语 | 8 条固定预设，前端硬编码 | 一键发送，零后端成本；后续按反馈调整 |
| 系统消息生成 | 后端显式调用 | 遵循后端权威原则，避免多端文案不一致 |
| 身份校验 | 信任 WS 连接生命周期 | 聊天不涉及筹码/公平性，WS room_id 绑定已足够 |
| 等待区玩家 | seat=-1，无额外限制 | 等待区也可发快捷短语，不增加判断逻辑 |

## 3. 快捷短语

前端硬编码常量，8 条固定列表：

| phrase_id | 中文 | 使用场景 |
|-----------|------|---------|
| 0 | 好牌！ | 称赞对手 |
| 1 | 运气不错 🍀 | 自己赢了 |
| 2 | All-in 了！ | 全压时刻 |
| 3 | 这把我弃了 | 弃牌 |
| 4 | 跟一手 | 跟注 |
| 5 | 加注！ | 加注 |
| 6 | GG 👍 | 结束礼貌 |
| 7 | 打得漂亮 | 称赞对手 |

## 4. WebSocket 协议

### 4.1 C2S — 玩家发送

统一使用 `chat` action，靠 `type` 区分消息类型：

```json
// 普通文本
{
  "action": "chat",
  "data": { "type": "text", "content": "这把打得不错" }
}

// 快捷短语
{
  "action": "chat",
  "data": { "type": "quick", "phrase_id": 0 }
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `action` | string | 固定 `"chat"` |
| `data.type` | string | `"text"` 或 `"quick"` |
| `data.content` | string | 文本内容（type=text 时必填），不做长度限制 |
| `data.phrase_id` | int | 快捷短语 ID 0-7（type=quick 时必填） |

### 4.2 S2C — 广播消息

```json
{
  "event": "chat_message",
  "data": {
    "seat": 3,
    "userId": "uuid",
    "nickname": "张三",
    "type": "text",
    "content": "这把打得不错",
    "timestamp": "2026-06-25T14:30:00"
  }
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `seat` | int | 发送者座位号：`-1` = 等待区，`0-9` = 座位号 |
| `userId` | string | 发送者用户 ID |
| `nickname` | string | 发送者昵称 |
| `type` | string | `"text"` / `"quick"` / `"system"` |
| `content` | string | 消息内容。text→用户输入原文，quick→前端查表渲染，system→后端格式化文案 |
| `timestamp` | string | ISO 8601 时间戳，后端生成，防止客户端时钟不准 |

### 4.3 新玩家加入 — 历史消息补发

新玩家建立 WebSocket 连接并通过 `auth` 认证后，后端下发缓存中最近 100 条消息：

```json
{
  "event": "chat_history",
  "data": {
    "messages": [
      { "seat": 0, "userId": "...", "nickname": "...", "type": "...", "content": "...", "timestamp": "..." },
      ...
    ]
  }
}
```

每条消息格式与 `chat_message` 的 `data` 完全一致。

## 5. 系统消息

### 5.1 触发事件与文案模板

| 触发事件 | type=system 的 content 模板 | 生成位置 |
|----------|---------------------------|---------|
| 玩家加入房间 | `"{nickname}" 加入了房间` | `main.py` — `join_room` |
| 玩家离开房间 | `"{nickname}" 离开了房间` | `main.py` — `leave_room` / WS 断连清理 |
| 玩家入座 | `"{nickname}" 坐上了 {seat}号位` | `room.py` — `seat_player` |
| 玩家离座回等待区 | `"{nickname}" 离开了 {seat}号位` | `room.py` — `unseat_player` |
| 玩家 Fold | `"{nickname}" 弃牌了` | `poker.py` — `fold()` |
| 玩家 Check | `"{nickname}" 过牌` | `poker.py` — `check()` |
| 玩家 Call | `"{nickname}" 跟注 {amount}` | `poker.py` — `call()` |
| 玩家 Bet | `"{nickname}" 下注 {amount}` | `poker.py` — `bet()` |
| 玩家 Raise | `"{nickname}" 加注到 {amount}` | `poker.py` — `raise_bet()` |
| 玩家 All-in | `"{nickname}" 全压了！` | `poker.py` — `all_in()` |
| 房主开始游戏 | `"游戏开始！"` | `main.py` — `start_game` |
| 牌局结束 | `"牌局结束"` | `state_machine.py` — `enter_reveal_phase` 或 `start_hand` 返回 uncontested 时 |

### 5.2 实现方式

在后端现有 `_emit` 广播游戏事件的地方，同时调用聊天广播函数：

```python
# 伪示例 — fold() 方法末尾
def fold(self, player: Player) -> None:
    player.is_folded = True
    # 原有游戏事件广播
    self._emit("player_folded", {"seat": player.seat})
    # 新增系统消息
    self._emit_chat_system(f'"{player.nickname}" 弃牌了')
```

`_emit_chat_system(content)` 封装为：
- 构造 `chat_message` 事件（`seat=-1, userId="", nickname="", type="system", ...`）
- 调用 `ConnectionManager.broadcast(room_id, event)`
- 同时写入最近 100 条内存缓存

## 6. 内存缓存

### 6.1 数据结构

每个房间维护一个固定大小的双端队列：

```python
from collections import deque

class ChatHistory:
    def __init__(self, max_size: int = 100):
        self._messages: deque[dict] = deque(maxlen=max_size)

    def append(self, message: dict) -> None:
        self._messages.append(message)

    def get_all(self) -> list[dict]:
        return list(self._messages)
```

### 6.2 生命周期

- 缓存挂载在 `RoomManager` 的房间实例上
- 房间创建时初始化，房间销毁时释放
- 内存占用估算：100 条 × 每条约 200 字节 = 20KB/房间，可忽略

## 7. 前端实现

### 7.1 组件结构

```
ChatPanel/              # 新增组件目录
├── ChatPanel.tsx       # 主面板（右侧可收起/展开）
├── ChatInput.tsx       # 输入框 + 发送按钮
├── QuickPhrases.tsx    # 快捷短语按钮组（2 行 × 4 列）
├── ChatMessage.tsx     # 单条消息渲染（区分 text/quick/system 样式）
└── ChatToggle.tsx      # 收起状态下的聊天图标按钮
```

### 7.2 ChatPanel 状态与行为

- **展开状态**：右侧固定宽度面板（约 280-320px），显示消息列表 + 输入区 + 快捷短语
- **收起状态**：右侧边缘一个小按钮/图标，有新消息时显示红点
- **消息排序**：最新在下，自动滚动到底部
- **消息样式**：
  - 自己发的 → 右对齐，蓝色气泡
  - 别人发的 → 左对齐，灰色气泡，头像 + 昵称
  - 系统消息 → 居中，浅灰小字
  - 快捷短语 → 类似普通消息但带特殊标记（如斜体）

### 7.3 WebSocket 集成

在 `useWebSocket.ts` 的消息处理中新增 `chat_message` 和 `chat_history` 事件处理：

- `chat_message`：追加到消息列表尾部
- `chat_history`：初始化消息列表（新玩家加入时）

### 7.4 防 XSS

React 默认会 escape 文本内容。不做 Markdown 渲染，纯文本展示，无需额外处理。

## 10. 消息卡片样式

### 普通文本消息

```
┌─ 自己发的 (右对齐) ──────────────────────────┐
│             这手牌打得不错  │ 14:30 │
└──────────────────────────────────────────────┘

┌─ 别人发的 (左对齐) ──────────────────────────┐
│ [头像] 张三                          14:30 │
│  运气不太好这把                          │
└──────────────────────────────────────────────┘
```

### 快捷短语

```
┌─ 左对齐（别人）/ 右对齐（自己）──────────────┐
│ 张三：好牌！                     14:30 │
└──────────────────────────────────────────────┘
```
前端收到 `type=quick` 时用 `phrase_id` 查本地常量表渲染文案。

### 系统消息

```
┌─ 居中 ──────────────────────────────────────┐
│         —— 张三 坐上了 3号位 ——            │
│         —— 游戏开始！ ——                  │
│         —— 张三 跟注 500 ——               │
└──────────────────────────────────────────────┘
```

## 11. 不做的

- 图片/表情/文件上传
- 私聊 / 观战频道 / 全局大厅聊天
- 聊天记录持久化到数据库
- 敏感词过滤
- 消息编辑/删除/撤回
- 快捷短语自定义
- Markdown 渲染
